# 文章大纲 (Outline)

## 1. 痛点：为什么还要造轮子？(Why)
*   **现有方案的困境**:
    *   **OneAPI/NewAPI (Go)**: 功能强大，但部署依赖多，二进制体积大，高并发下 GC 可能导致延迟抖动。
    *   **Java/Python 方案**: 内存占用动辄 500MB+，对小配置 VPS 不友好。
    *   **UI 丑陋**: 大多数开源项目的后台管理界面停留在 Bootstrap 时代，缺乏美感。
*   **我的执念**:
    *   **极致性能**: 能用 10MB 内存解决的事，绝不用 100MB。
    *   **原生体验**: 想要一个像 Windows 11 原生应用一样丝滑的管理界面，而不是卡顿的网页。
    *   **All in Rust**: 验证 Rust 在高并发网关层面的统治力。

## 2. BurnCloud 是什么？(What)
*   **一句话介绍**: 一个 **Rust 原生**、**高性能**、**本地优先** 的 LLM 聚合网关与管理平台。
*   **核心特性 (Showcase)**:
    *   🚀 **Rust 驱动**: 基于 Axum + Tokio，单核万级并发。
    *   ⚡ **零损耗透传**: 独创 "Don't Touch the Body" 模式，打字机效果零延迟。
    *   🎨 **Fluent Design**: 基于 Dioxus (Rust GUI) 构建的现代化管理界面。
    *   🇨🇳 **国产友好**: 首发支持 DeepSeek、Qwen、Yi 等国产模型，内置多线路优化。
