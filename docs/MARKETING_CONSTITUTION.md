# BurnCloud 营销与品牌宪法 (Marketing & Brand Constitution)

**版本**: 1.0
**生效日期**: 2025-12-06

## 序言

技术是 BurnCloud 的骨架，而营销是它的血肉。为了让 BurnCloud 成为全球开发者首选的大模型聚合与管理平台，我们在坚持“代码如诗”的同时，也必须奉行“营销即服务”的理念。本宪法确立了项目的市场定位、传播原则和增长策略。

---

## 第一章：核心品牌定位 (Brand Identity)

### 1.1 品牌口号 (Slogan)
**中文**: "奔云 - 重塑你的 AI 基础设施"
**English**: "BurnCloud - Rewrite Your AI Infrastructure."

### 1.2 核心价值主张 (Value Proposition)
*   **Rust Native**: 强调高性能、低资源占用、无 GC 停顿。这是我们区别于 Java/Go/Python 竞品的最大护城河。
*   **Privacy First**: 本地优先，数据在我。直击企业和极客对隐私泄露的痛点。
*   **One for All**: 一个二进制文件，统管所有模型。极致的部署体验。

### 1.3 目标受众 (Target Audience)
1.  **独立开发者/Indie Hackers**: 需要低成本、高性能的 API 网关来构建自己的 AI 应用。
2.  **中小企业技术团队**: 需要统一管理公司内部对 OpenAI/Claude/Gemini 的访问权限和额度。
3.  **Rust 爱好者**: 渴望看到 Rust 在生产级 Web 服务/网关领域的杀手级应用。

---

## 第二章：内容营销原则 (Content Strategy)

### 2.1 讲故事，不只是列功能 (Storytelling)
*   **拒绝**: "我们支持负载均衡功能。"
*   **提倡**: "当 GPT-4 API 突然宕机时，你的业务会停摆吗？BurnCloud 的智能故障转移机制，让你的服务像不死鸟一样由其他渠道无缝接管，用户甚至感觉不到发生了什么。"

### 2.2 透明化开发 (Building in Public)
*   鼓励在 Twitter/X, Reddit (r/rust, r/selfhosted), 掘金, V2EX 上分享开发过程中的**技术挑战与解决方案**。
*   **示例**: "我们在用 Rust 重写网关鉴权层时，通过消除一次多余的内存拷贝，将吞吐量提升了 300%。这是具体的 Benchmark..." (硬核技术文是最好的软广)。

### 2.3 视觉美学 (Visual Aesthetics)
*   所有的 Demo 截图、架构图必须精美、现代。
*   强调 UI 的 **Fluent Design**，打破人们对“开源软件界面简陋”的刻板印象。

## 第三章：增长黑客策略 (Growth Hacking)

### 3.1 SEO 关键词矩阵
*   **Core**: `Rust LLM Gateway`, `OpenAI API Aggregator`, `Self-hosted AI Proxy`
*   **Long-tail**: `OneAPI alternative in Rust`, `High performance AI gateway`, `Gemini to OpenAI format converter`

### 3.2 社区渗透
*   **GitHub**: 优化 `README.md` (已开始)，提供一键部署 Docker 镜像，争取登上 Trending。
*   **Hacker News**: 准备高质量的 Show HN 帖子，重点突出 "Rust" 和 "Performance"。

---

## 第四章：道德营销准则 (Ethical Standards)

*   **实事求是**: 严禁夸大性能指标。Benchmark 必须可复现。
*   **尊重竞品**: 可以对比技术架构的优劣 (如 Rust vs Go 的内存占用)，但严禁通过贬低竞品的人格或团队来抬高自己。
*   **隐私承诺**: 营销活动中收集的任何用户邮箱（如 Newsletter），严禁出售或滥用。
